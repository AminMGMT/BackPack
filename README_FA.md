# بک‌پک 🎒

**بک‌پک** یک هسته‌ی تونل معکوس (Reverse Tunnel) با کارایی بالاست که کاملاً با **Go**
نوشته شده و برای ست‌آپ سرور ایران ⇄ خارج طراحی شده. یک باینری واحد است با یک منوی
تعاملی CLI **و** یک پنل وب امن — یعنی همه‌چیز را با ترمینال یا بدون ترمینال می‌توانی
مدیریت کنی.

> TeleGram: **@BlackProtocols** · GitHub: **[AminMGMT/BackPack](https://github.com/AminMGMT/BackPack)**
>
> 📖 English guide: **[README.md](README.md)**

---

## امکانات

- **ترنسپورت:** TCP و UDP، تونل معکوس با Connection Pool.
- **پریست Best Performance:** با یک انتخاب همه‌چیز تیون می‌شود (nodelay، پول بزرگ،
  بافر سوکت ۸ مگابایتی، BBR و تیونینگ کرنل) برای کمترین لتنسی و بیشترین سرعت.
- **منوی CLI:** ساخت تونل، وضعیت زنده، مدیریت، بهینه‌سازی، آپدیت.
- **پنل وب (پورت ۷۷۷۷):** لاگین‌دار، تم دارک با رنگ‌های قابل‌تغییر، نمایش زنده‌ی
  CPU/RAM/دیسک/ترافیک، وضعیت + پینگ + لاگ هر تونل، و می‌توانی **تونل بسازی و مدیریت
  کنی، تلگرام را تنظیم کنی، بازه‌ی Auto Refresh و پسورد را عوض کنی — همه از داخل مرورگر.**
- **گزارش تلگرام:** ارسال دوره‌ای وضعیت؛ اگر سرور به تلگرام دسترسی نداشت (مثل ایران)،
  از طریق یک تونل و SOCKS5 داخلی رله می‌کند.
- **Auto Refresh:** ری‌استارت همه‌ی تونل‌ها هر N ساعت.
- **آپدیت تک‌کلیکی** با git pull از ریپوی گیت‌هاب خودت.
- **واچ‌داگ خودترمیم:** تونل drop‌شده (هر طرف) را تشخیص می‌دهد و در حدود ۱ دقیقه خودش ری‌استارت می‌کند.
- سرویس‌های **systemd** که بعد از ریبوت و بستن ترمینال زنده می‌مانند.

## کدام سرور Server است و کدام Client؟ (مهم‌ترین نکته)

| سرور | نقش | گزینه‌ی منو | چرا |
|------|-----|------------|-----|
| **سرور ایران** | ورودی | **Setup Server** | پورت‌ها را expose می‌کند؛ کاربر به **IP ایران** وصل می‌شود (سریع و بدون فیلتر). |
| **سرور خارج** | خروجی | **Setup Client** | به سرور ایران dial می‌کند و ترافیک را به سرویس واقعی می‌رساند. |

```
   کاربر ──▶  سرور ایران (SERVER، پورت‌ها را expose می‌کند)  ──تونل──▶  خارج (CLIENT، سرویس واقعی)
```

**همیشه اول سرور ایران (Server) را ست کن، بعد سرور خارج (Client) را.** کلاینت به آدرس
ایران و توکنی که سرور می‌سازد نیاز دارد.

## نصب

روی VPS با کاربر root. چون **گیت‌هاب از ایران بلاکه**، دو طرف آدرس clone متفاوت دارند:

### سرور خارج — مستقیم:

</div>

```bash
git clone https://github.com/AminMGMT/BackPack.git && cd BackPack && sudo bash install.sh && sudo backpack
```

<div dir="rtl">

### سرور ایران — از طریق پروکسی گیت‌هاب (اسم فولدر همان `BackPack` می‌ماند):

</div>

```bash
git clone https://gh-proxy.com/https://github.com/AminMGMT/BackPack.git BackPack && cd BackPack && sudo bash install.sh && sudo backpack
```

<div dir="rtl">

`install.sh` با **میرورهای در دسترس از ایران** build می‌کند: تول‌چین Go از **Aliyun**، و
ماژول‌های Go از پروکسی ایرانی **RunFlare** (`mirror-go.runflare.com`) — پس برای build نیازی
به دسترسی مستقیم به گیت‌هاب نیست. `GOTOOLCHAIN=local` هم ست می‌شود. خودش حالت را تشخیص می‌دهد:

۱. **باینری آماده‌ی محلی** (`./backpack`، `dist/`، `prerequisite/`) → همان را نصب می‌کند.
۲. **سورس موجود** (`go.mod`) → build می‌کند.

> **آپدیت از ایران:** وقتی یک تونل بالا باشد، دکمه‌ی **Update** دستور `git pull` را از طریق
> رله‌ی SOCKS تونل به سرور خارج می‌فرستد (که به گیت‌هاب دسترسی دارد)، پس آپدیت حتی با بلاک‌بودن
> گیت‌هاب هم کار می‌کند. (نیازمند تونلی که پورت رله‌ی SOCKS داشته باشد — مثل همان که برای ربات
> تلگرام ست شده.) در غیر این صورت از remoteِ `gh-proxy.com` استفاده می‌کند.

### نصب کاملاً آفلاین

روی یک ماشین با اینترنت `bash prerequisite/download-prerequisites.sh` را اجرا کن تا
تول‌چین Go + دیپندنسی‌ها + باینری آماده داخل فولدر جمع شود؛ کل فولدر را به VPS آفلاین
ببر و `sudo bash install.sh` بزن. جزئیات در [`prerequisite/README.md`](prerequisite/).

## شروع سریع

### ۱) روی سرور ایران — ساخت تونل Server

**روش الف — از CLI:**

</div>

```bash
sudo backpack   →  1. Setup Server
```

<div dir="rtl">

ترنسپورت (TCP/UDP)، پورت تونل و پورت‌های expose را بده، **توکن ۶۴ کاراکتری** پیشنهادی
را با Enter بپذیر، و پریست **Best Performance** را انتخاب کن. توکن را کپی کن (برای کلاینت لازم است).

**روش ب — از پنل وب:** آدرس `http://<ip-ایران>:7777` را باز کن، لاگین کن، **+ Add Tunnel**
را بزن، اسم/پورت/ترنسپورت/پورت‌های expose را پر کن → **Create**. توکن برایت نمایش داده می‌شود.

### ۲) روی سرور خارج — ساخت تونل Client

</div>

```bash
sudo backpack   →  2. Setup Client
```

<div dir="rtl">

**IP سرور ایران**، پورت تونل، و **همان توکن** را وارد کن. تمام.

## پنل وب (پورت ۷۷۷۷)

با ساخت اولین تونل خودکار بالا می‌آید (سرویس systemd به اسم `backpack-webui`، پس با بستن
ترمینال هم زنده می‌ماند). آدرس `http://<ip-سرور>:7777`.

- **ورود** با **رمز ۸ رقمی** که خودکار ساخته می‌شود (یا رمز دلخواه خودت).
- **متریک زنده:** حلقه‌های CPU/RAM/دیسک/Swap، سرعت آپلود/دانلود، ترافیک کل، Uptime، Load،
  hostname، IPv4/IPv6، و **موقعیت و ISP سرور**.
- **تونل‌ها:** هر کارت وضعیت (online/offline/stopped)، پینگ، پورت‌ها، **پرچمِ کشور**،
  موقعیت/ISP سرور مقابل، و **View Logs** را دارد — به‌علاوه Start/Stop/Restart/Delete.
- **+ Add Tunnel:** ساخت تونل Server از مرورگر — اسم، پورت، ترنسپورت، پورت‌های expose،
  **کشور**، و یک **پورت SOCKS5 دلخواه** (اختیاری) برای رله‌ی تلگرام. توکن ساخته‌شده با کلیک کپی می‌شود.
- **آیکون‌های هدر:** 💬 Support (لینک‌ها + دونیت)، تنظیم ربات تلگرام، و ⚙️ Settings
  (رنگ تم، **Update** تک‌کلیکی، **Auto Refresh**، **تغییر رمز**).

> برای دسترسی، پورت ۷۷۷۷ را در فایروال باز کن: `sudo ufw allow 7777`.

## Best Performance

با فعال‌کردن پریست، مقادیر بهینه پر می‌شوند و تو فقط پایه را می‌دهی: `nodelay=true`،
`connection_pool=8`، `channel_size=4096`، بافر سوکت **۸ مگابایتی** (پایپ را روی مسیر پرلتنسی
پر می‌کند → سرعت بالاتر)، و **Optimize** را اجرا می‌کند (BBR + fq، سقف بافر ۲۵۶ مگابایت،
TCP Fast Open، لیمیت فایل ۱ میلیون). Aggressive pool عمداً خاموشه تا CPU در حالت idle پایین بماند.

## ربات تلگرام

از CLI (**Telegram Bot**) یا پنل وب (**Settings → Telegram bot**) تنظیمش کن: توکن بات
(از @BotFather)، آیدی ادمین (از @userinfobot)، و بازه‌ی زمانی. گزارش‌ها آدرس پنل + رمز را هم دارند.

اگر سرور **به تلگرام دسترسی ندارد** (مثل ایران)، گزینه‌ی **ارسال از طریق یک تونل** را انتخاب کن:
بک‌پک یک پورت random روی آن تونل باز می‌کند که به یک **SOCKS5** داخلی روی سرور خارج map می‌شود
(احراز هویت با توکن مشترک تونل)، و تلگرام را از داخل تونل رد می‌کند — **بدون نیاز به باز کردن
هیچ پورت اضافه‌ای روی فایروال**. بعدش یک‌بار تونل کلاینت را ری‌استارت کن که پورت جدید را بگیرد.

بات **تعاملی** است — هر پیام سه دکمه دارد:

- **📊 Status** — وضعیت لحظه‌ای همه‌ی سرورها/تونل‌ها.
- **🖥 Web UI** — آدرس و رمز پنل.
- **💬 Support** — گیت‌هاب، کانال تلگرام، و آدرس‌های حمایت.

## آپدیت

**Update** (گزینه‌ی ۹ منو، یا پنل وب → Settings → Update) روی ریپوی clone‌شده `git pull`
می‌زند، دوباره build می‌کند و سرویس‌ها را ری‌استارت می‌کند. کافیه تغییراتت را به گیت‌هاب push
کنی و روی سرور دکمه‌ی Update را بزنی.
