# بک‌پک 🎒

**بک‌پک** یک هسته‌ی تونل معکوس (Reverse Tunnel) با کارایی بالاست که کاملاً با **Go**
نوشته شده و برای ست‌آپ سرور ایران ⇄ خارج طراحی شده. یک باینری واحد است با یک منوی
تعاملی CLI **و** یک پنل وب امن — یعنی همه‌چیز را با ترمینال یا بدون ترمینال می‌توانی
مدیریت کنی.

> TeleGram: **@BlackProtocols** 
>
> 📖 English guide: **[README.md](README.md)**

---

## امکانات

- **ترنسپورت:** TCP، TCP Mux، UDP، WS، WS Mux، WSS و WSS Mux — تونل معکوس با Connection Pool (برای WSS گواهی self-signed خودکار ساخته می‌شود).
- **پریست Best Performance:** با یک انتخاب همه‌چیز تیون می‌شود (nodelay، پول بزرگ،
  بافر سوکت ۸ مگابایتی، BBR و تیونینگ کرنل) برای کمترین لتنسی و بیشترین سرعت.
- **منوی CLI:** ساخت تونل، **ویرایش پورت‌ها و ترنسپورت**، وضعیت زنده، **بررسی
  سلامت**، بکاپ/ری‌استور، تلگرام، بهینه‌سازی، آپدیت — همه از همین‌جا.
- **فِیل‌اوور خودکار:** کلاینت می‌تواند چند **آدرس پشتیبان** داشته باشد (IP دوم،
  پورت دیگر، یا edge یک CDN)؛ اگر آدرس اصلی جواب نداد خودش می‌رود سراغ بعدی —
  یعنی وقتی IP سرور فیلتر شد، تونل بدون هیچ کاری از طرف تو بالا می‌ماند.
- **هیچ‌وقت خودش را خراب نمی‌کند:** آپدیت و ویرایش قبلش نقطه‌ی بازگشت می‌سازند،
  نتیجه را چک می‌کنند و اگر تونل بالا نیامد **خودکار برمی‌گردند عقب**.
- **بررسی سلامت:** یک صفحه که سرور، پنل و همه‌ی تونل‌ها را تست می‌کند و برای هر
  مشکل دقیقاً می‌گوید چه‌کار کنی.
- **پنل وب مانیتورینگ (پورت ۷۷۷۷):** لاگین‌دار، تم دارک، نمایش زنده‌ی
  CPU/RAM/دیسک/ترافیک و وضعیت + پینگ + لاگ هر تونل. فقط برای **مانیتورینگ** است و
  بهتره روی **سرور ایران** ازش استفاده کنی. (تنظیمات: تم، آپدیت، پورت پنل، رمز.)
- **گزارش تلگرام از داخل تونل:** ارسال دوره‌ای وضعیت که از طریق یک تونل و SOCKS5
  داخلی رله می‌شود — چون سرور ایران به تلگرام دسترسی ندارد، این روش ۱۰۰٪ کار می‌کند.
- **Auto Refresh:** ری‌استارت همه‌ی تونل‌ها هر N ساعت (زیر منوی **Manage**).
- **آپدیت بر پایه‌ی ریلیز:** با یک کلیک آخرین ریلیز گیت‌هاب دانلود و جایگزین می‌شود
  (مستقیم → رله‌ی تونل → میرورها) — بدون نیاز به Go یا git روی سرور.
- **بکاپ و ری‌استور کامل:** همه‌ی تونل‌ها، رمز پنل، تنظیمات تلگرام، گواهی‌های TLS
  و زمان‌بندی Auto Refresh در یک فایل `.tar.gz` در پوشه‌ی `/root/BackPack/backups`.
- **واچ‌داگ خودترمیم:** تونل drop‌شده (هر طرف) را تشخیص می‌دهد و در حدود ۱ دقیقه خودش ری‌استارت می‌کند.
- سرویس‌های **systemd** که بعد از ریبوت و بستن ترمینال زنده می‌مانند.

## کدام سرور Server است و کدام Client؟ (مهم‌ترین نکته)

| سرور | نقش | گزینه‌ی منو | چرا |
|------|-----|------------|-----|
| **سرور ایران** | ورودی | **Setup Server** | پورت‌ها را expose می‌کند؛ کاربر به **IP ایران** وصل می‌شود (سریع و بدون فیلتر). |
| **سرور خارج** | خروجی | **Setup Client** | به سرور ایران dial می‌کند و ترافیک را به سرویس واقعی می‌رساند. |

```
   کاربر ──▶  سرور ایران (SERVER، پورت‌ها را expose می‌کند)  ──تونل──▶  خارج (CLIENT، سرویس واقعی)
```

**همیشه اول سرور ایران (Server) را ست کن، بعد سرور خارج (Client) را.** کلاینت به آدرس
ایران و توکنی که سرور می‌سازد نیاز دارد.

## نصب

فقط یک دستور با کاربر root روی VPS — فایل **tar.gz ریلیز** مخصوص معماری سرورت
(amd64/arm64) در **`/root/BackPack`** دانلود و باینری نصب می‌شود. اول **مستقیم** از
گیت‌هاب امتحان می‌کند و اگر نشد سراغ میرورهای در دسترس از ایران می‌رود:

</div>

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/AminMGMT/BackPack/main/install.sh)
```

<div dir="rtl">

### سرور ایران — اگر raw گیت‌هاب بلاک بود، خود اسکریپت را از پروکسی بگیر:

</div>

```bash
bash <(curl -fsSL https://gh-proxy.com/https://raw.githubusercontent.com/AminMGMT/BackPack/main/install.sh)
```

<div dir="rtl">

بعدش منو را باز کن:

</div>

```bash
sudo backpack
```

<div dir="rtl">

همه‌چیز مرتب و در یک‌جا: باندل ریلیز در `/root/BackPack`، بکاپ‌ها در
`/root/BackPack/backups`، کانفیگ تونل‌ها در `/etc/backpack`.

> **build از سورس** به‌عنوان راه دوم هنوز کار می‌کند: ریپو را clone کن و داخلش
> `sudo bash install.sh` بزن — اگر دانلود ریلیز شکست بخورد، با Go build می‌کند
> (ماژول‌ها اول **مستقیم**، و فقط اگر نشد از میرورهای RunFlare/goproxy.cn).

### نصب آفلاین (وقتی سرور اینترنت درست‌وحسابی ندارد)

روی هر سیستمی که اینترنت دارد، این دو فایل را از
[صفحه‌ی ریلیزها](https://github.com/AminMGMT/BackPack/releases/latest) دانلود کن:

۱. `install.sh`
۲. آخرین `backpack_linux_amd64.tar.gz` (یا `..._arm64.tar.gz` — معماری سرور را با
`uname -m` چک کن؛ از ایران می‌توانی اول آدرس دانلود `https://gh-proxy.com/` بگذاری)

هر دو را در **یک فولدر** روی VPS کپی کن و نصاب را اجرا کن — خودش فایل tar.gz
محلی را پیدا می‌کند و هیچ اینترنتی روی سرور لازم نیست:

</div>

```bash
scp install.sh backpack_linux_amd64.tar.gz root@SERVER_IP:/root/
ssh root@SERVER_IP "cd /root && sudo bash install.sh"
```

## شروع سریع

### ۱) روی سرور ایران — ساخت تونل Server

</div>

```bash
sudo backpack   →  1. Setup Server
```

<div dir="rtl">

ترنسپورت (TCP/TCPMux/UDP/WS/WSS/WSMux/WSSMux)، پورت تونل و پورت‌های expose را بده، **توکن ۶۴ کاراکتری** پیشنهادی
را با Enter بپذیر، و پریست **Best Performance** را انتخاب کن. توکن را کپی کن (برای کلاینت لازم است).

### ۲) روی سرور خارج — ساخت تونل Client

</div>

```bash
sudo backpack   →  2. Setup Client
```

<div dir="rtl">

**IP سرور ایران**، پورت تونل، و **همان توکن** را وارد کن. تمام.

## مرور سریع

- **منوی CLI (`sudo backpack`):** ساخت Server / Client · **Manage** (برای هر
  تونل **Edit**: پورت‌ها، ترنسپورت و آدرس‌های پشتیبان، Start/Stop/Restart، لاگ
  زنده، حذف · وضعیت زنده · **Health Check** · ری‌استارت همه · Auto Refresh ·
  **File Locations**) · **Backup & Restore** · **Web Panel** · Optimize ·
  Telegram Bot · **Update** (با نقاط بازگشت) · Uninstall. کنار هر گزینه یک
  توضیح کوتاه خاکستری هست.
- **آدرس‌های پشتیبان:** روی کلاینت از **Edit → Backup server addresses** چند
  آدرس اضافه کن (`1.2.3.4, 5.6.7.8:8443, edge.example.com:443`). اگر آدرس اصلی
  جواب ندهد خودکار می‌رود سراغ بعدی — همان چیزی که بعد از فیلتر شدن IP سرور
  تونل را زنده نگه می‌دارد.
- **Health Check و File Locations:** زیر منوی **Manage**. بررسی سلامت، تیونینگ
  کرنل، پنل، وضعیت هر تونل، دسترسی واقعی TCP، انقضای گواهی TLS و قدرت توکن را
  چک می‌کند و زیر هر مشکل راه‌حلش را می‌نویسد.
- **پنل وب (پورت ۷۷۷۷):** داشبورد **فقط مانیتورینگ** — CPU/RAM/دیسک/ترافیک
  زنده، وضعیت + پینگ واقعی + لاگ هر تونل. بهتره روی **سرور ایران** باشد؛ لینک و
  رمز ورود در CLI زیر بخش **Web Panel** است (تنظیمات: تم، آپدیت، پورت پنل، رمز).
  اول پورت را باز کن: `sudo ufw allow 7777`.
- **ربات تلگرام:** گزارش دوره‌ای وضعیت که **از داخل یک تونل** ارسال می‌شود — یک
  پورت random روی تونل به SOCKS5 داخلیِ سرور خارج map می‌شود، پس از ایران ۱۰۰٪
  کار می‌کند. دکمه‌های تعاملی: Status / Web UI / Support.
- **پریست Best Performance:** مقادیر بهینه (nodelay، پول، بافر سوکت ۸ مگابایت)
  را پر می‌کند و تیونینگ کرنل (BBR + fq، سقف بافرها، لیمیت فایل) را اعمال می‌کند.
- **بکاپ و ری‌استور:** همه‌چیز (تونل‌ها + توکن‌ها، رمز پنل، تلگرام، گواهی‌های TLS،
  زمان‌بندی) در یک فایل `.tar.gz` در `/root/BackPack/backups`؛ ری‌استور همه‌ی
  تونل‌ها را دوباره می‌سازد و استارت می‌کند.
- **آپدیت:** گزینه‌ی **Update** خودش تشخیص می‌دهد ریلیز جدیدتری روی گیت‌هاب هست و
  فایل آماده‌ی `backpack_linux_<arch>.tar.gz` را نصب می‌کند (مستقیم → رله‌ی تونل
  → میرورها — از ایران هم کار می‌کند). قبلش یک **نقطه‌ی بازگشت** می‌سازد، بعدش
  سلامت را چک می‌کند و اگر چیزی بالا نیامد خودکار برمی‌گردد عقب؛ دستی هم از
  **Update → Restore points** می‌توانی برگردی. اگر قبلاً با clone نصب کرده‌ای
  (≤ v1.2.0) فقط یک‌بار Update بزن؛ از آن به بعد خودکار از ریلیزها آپدیت می‌شود.
- **چیدمان روی سرور:** باندل ریلیز + بکاپ‌ها در `/root/BackPack`، کانفیگ تونل‌ها
  در `/etc/backpack`، باینری در `/usr/local/bin/backpack`.
